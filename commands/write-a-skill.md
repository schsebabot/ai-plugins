# Write a Skill Command

Follow the `write-a-skill` skill from this plugin.

Use the text after the command name as the skill description or requirements. If your command runtime exposes `$ARGUMENTS`, treat it as that same value.

Read and follow:
- `skills/write-a-skill/SKILL.md`

If the user did not describe what skill to create, ask them about the domain, use cases, and triggers before proceeding. Walk through the full workflow: gather requirements, create the skill directory and files, create the command file, update README.md, and run the review checklist.
